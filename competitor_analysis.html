01 r_keeper 7

🍽
r_keeper 7 (UCS)
Российская POS-система #1 в СНГ. Используется в тысячах ресторанов, в т.ч. в Армении.

✅ Что умеет

  • 🍽 Полная автоматизация ресторана (POS, касса, кухня)
  • 📱 QR-меню как дополнительный модуль (партнёры: Carbis, SkyMenu)
  • 🔗 XML/JSON API для получения актуального меню
  • 🛑 Синхронизация стоп-листов в реальном времени
  • 📦 Модификаторы, комбо-блюда, расписания
  • 💳 RkPayQR — оплата через QR прямо в системе

❌ Слабые стороны

  • 💸 QR-меню — платный модуль, сложная настройка
  • 🌍 Нет нормального мультиязычного меню для туристов
  • 📊 Аналитика меню слабая (нет "что смотрели", только что заказали)
  • 🎨 Шаблоны меню некрасивые, нет кастомизации
  • 🇦🇲 Нет локализации под Армению, армянского языка нет
  • 🚀 Долгое внедрение, нужен интегратор

⚠️ Угроза: СРЕДНЯЯ

r_keeper — это тяжёлая POS-система, а не лёгкое меню. Рестораны, которые его используют, уже платят за него ~$100-300/мес и не будут переходить. Но у них есть боль — некрасивое QR-меню без армянского языка. Это наш шанс.

💡
Возможность: Предложить r_keeper-ресторанам TapMenu как красивый "фронтенд для меню" поверх их системы — через интеграцию по API. Им не нужно менять кассу. Им просто нужно красивое армяноязычное меню для туристов.

02 iiko

🤖
iiko (Paloma365)
Второй по популярности POS в СНГ. Активно используется в Армении. Есть собственный App Store для плагинов.

✅ Что умеет

  • 🍽 Полный POS + управление кухней + склад
  • 📱 QR-меню встроенное (iiko Web)
  • 🏪 Собственный App Store (iiko.store) — плагины от партнёров
  • 🔗 REST API — получение меню, стоп-листы, заказы
  • 📲 Smartofood интегрирован — сайт + QR + приложение
  • 🌍 Есть мультиязычность (частичная)

❌ Слабые стороны

  • 🇦🇲 Армянский язык — не поддерживается нативно
  • 💱 Конвертация валют — нет (AMD/USD/EUR)
  • 🎨 Визуальные шаблоны стандартные, нет премиум-дизайна
  • 💸 Полный стек iiko + QR-меню = дорого ($200+/мес)
  • 🏨 Турист-мод: нет Wi-Fi кнопки, нет tourist mode
  • 🤖 AI-функций нет вообще

⚠️ Угроза: СРЕДНЯЯ-ВЫСОКАЯ

iiko активно работает в Армении. У них есть API и экосистема плагинов. Но их QR-меню — функциональное, не красивое. И у них нет армянского языка, нет tourist mode, нет AI. Наш продукт выигрывает по UX и локализации.

🔌
Партнёрская идея: Опубликоваться в iiko.store как плагин "TapMenu — красивое мультиязычное меню для гостей". Тогда iiko-рестораны сами найдут нас внутри их системы.

03 Smartofood

🛒
Smartofood
Российский SaaS: сайт доставки + QR-меню + мобильное приложение. Глубокая интеграция с iiko. ~834₽/мес за QR.

✅ Что умеет

  • 🔗 Синхронизация с iiko в реальном времени
  • 💳 Онлайн-оплата через QR-меню
  • 📱 Telegram Mini-App
  • 📊 Аналитика заказов
  • 🛑 Стоп-лист автоматически
  • 💸 Низкая цена ~$9/мес

❌ Слабые стороны

  • 🇦🇲 Нет армянского языка, не заточен под Армению
  • 🌍 Нет tourist mode / мультивалютности
  • 🎨 Шаблоны простые, нет кастомного брендинга
  • 🤖 Нет AI-функций
  • 🇷🇺 Российский продукт — санкционные риски для армянских банков
  • 📡 Только iiko — нет поддержки r_keeper / Poster

🟡 Угроза: НИЗКАЯ для Армении

Smartofood ориентирован на российский рынок и iiko-экосистему. В Армении работает, но не локализован. Плюс — политические и банковские риски использования российских SaaS платформ в Армении. TapMenu — армянский продукт с армянским языком и AMD-ценами.

04 Poster POS + QR

📋
Poster POS (joinposter.com)
Украинский облачный POS. Есть встроенный QR-модуль. Популярен в небольших кафе СНГ.

✅ Что умеет

  • ☁️ Облачный POS — касса + склад + CRM
  • 📱 QR-меню встроенное
  • 💳 Онлайн-заказы и оплата
  • 💸 Доступные тарифы от $25/мес
  • 🔗 Открытый API

❌ Слабые стороны

  • 🇦🇲 Нет армянского языка и локализации AMD
  • 🌍 Tourist mode — нет
  • 🎨 Базовые шаблоны, нет премиум-вида
  • 🤖 AI — нет
  • 🇺🇦 Украинский продукт — нестабильный доступ к поддержке

🟡 Угроза: НИЗКАЯ

Poster — хороший продукт, но не адаптирован под Армению. Для малых кафе в Армении он избыточен как POS. TapMenu не конкурирует с POS — мы делаем лучший "фасад меню" для гостей.

05 MENU TIGER / TableQR

🐯
MENU TIGER / TableQR / iMenuPro
Западные SaaS QR-меню. Глобальные, без локализации для СНГ/Армении.

✅ Что умеет

  • 🌍 Мультиязычность (много языков)
  • 🎨 Красивые шаблоны меню
  • 💳 Некоторые поддерживают онлайн-оплату
  • 📊 Базовая аналитика
  • 💸 От $10-30/мес

❌ Слабые стороны

  • 🇦🇲 Нет армянского языка (HY)
  • 💱 Нет AMD валюты
  • 🔌 Нет интеграции с r_keeper/iiko
  • 💳 Stripe-only — нет IDram
  • 🤖 Нет AI-ассистента
  • 📞 Поддержка только на английском

🟢 Угроза: ОЧЕНЬ НИЗКАЯ — это наша точка отличия

Западные конкуренты физически не могут предложить армянский язык, AMD и локальные платёжные системы. TapMenu — единственный армяноязычный продукт с нативной поддержкой всего местного.

Американские POS-гиганты с QR-заказами. Не работают в Армении.

  • 💳 Visa/MC only — нет IDram, нет AMD
  • 💸 $100-500/мес + оборудование
  • 📞 Нет армянской поддержки

🟢 Угроза: НУЛЕВАЯ в ближайшие 3-5 лет

Toast и Square физически не представлены в Армении. Их не будет здесь ещё долго — слишком маленький рынок для них. Наше окно возможностей — несколько лет.

08 Идея: интеграция с POS-системами

🤔
Вопрос: Стоит ли интегрироваться с r_keeper / iiko? Это даст доступ к ресторанам, у которых уже есть POS. Они не будут менять кассу — им нужно только красивое меню поверх.

Сценарий А: TapMenu как "красивый фронтенд" поверх POS

✅ Что даёт интеграция
  • 🔄 Меню синхронизируется автоматически из r_keeper/iiko
  • 🛑 Стоп-лист обновляется в реальном времени
  • 📋 Не нужно вводить блюда вручную — всё уже в POS
  • 🎯 Большой рынок: все r_keeper/iiko рестораны Армении
  • 💰 Более высокая цена — интеграция = ценность
❌ Риски и сложности
  • 🔧 Техническая сложность: XML/JSON API r_keeper нестабилен
  • ⏰ Долго: минимум 4-8 недель разработки
  • 🔒 r_keeper требует платный API-ключ для партнёров
  • 🐛 При обновлении POS-системы интеграция может сломаться
  • 🎯 Отвлекает от MVP — риск не выпустить вовремя
📅
Рекомендация: Интеграции — не в MVP, а в Phase 3 (после Phase 2 с платежами).

Приоритет интеграций:
  1. iiko API — более открытый, популярнее в Армении, есть App Store
  2. r_keeper API — сложнее, но охват больше
  3. Poster API — для малых кафе, самый простой API

Абстрактный слой источника меню

В схеме БД добавить поле menu_source в таблицу restaurants: 'manual' | 'iiko' | 'rkeeper' | 'poster'. Сейчас всегда 'manual', но архитектура готова к интеграции без переписывания.

🏗 Схема БД · Добавить в ТЗ

Поля для POS-интеграции в таблице restaurants

Зарезервировать поля: pos_system VARCHAR, pos_api_key TEXT (encrypted), pos_venue_id VARCHAR, pos_last_sync_at TIMESTAMP. В MVP — NULL, в Phase 3 — заполняются.

09 Идея: Smart Brain ресторана

🧠
Концепция: Вместо просто "цифрового меню" — позиционировать TapMenu как "умный мозг ресторана", который не только показывает меню, но и помогает управлять бизнесом через AI-аналитику.

Что уже есть в TapMenu из "Smart Brain"

  • ✅ Аналитика просмотров по блюдам, времени, языкам
  • ✅ AI-ассистент (GPT-4o) для LUXE — советы, описания, фото
  • ✅ AI-анализ аналитики: "почему вечером мало гостей?"
  • ✅ Стоп-лист в реальном времени
  • ✅ Спецпредложения по времени (бизнес-ланч 12:00-15:00)

Что можно добавить для "Smart Brain" (Phase 2-3)

🧠 AI Insights — умные советы
  • 📈 "Хит этой недели — Долма, +40% просмотров"
  • ⏰ "Пик в 13:00-14:00 — добавьте бизнес-ланч"
  • 🌍 "70% гостей — русскоязычные, добавьте RU-описания"
  • 📸 "У 5 блюд нет фото — это снижает клики на 60%"
  • 💡 "Ваш стейк не смотрят — попробуйте изменить описание"
📊 Предиктивная аналитика
  • 📅 Прогноз загруженности по дням недели
  • 🌦 Корреляция погоды и посещаемости (API погоды)
  • 🎉 Предупреждение о праздниках (армянский календарь)
  • 💱 Всплески туристов — предложить перевод на новый язык
📲 WhatsApp / Telegram интеграция
  • 📩 Ежедневный отчёт ресторану в WhatsApp
  • 🛑 Мгновенное уведомление: "Топ-3 блюда сегодня"
  • ⚠️ Алерт: "Подписка истекает через 3 дня"
  • 🤖 Telegram-бот для управления стоп-листом
⭐ Отзывы и репутация
  • ⭐ Быстрая оценка блюда прямо в меню (1 клик)
  • 📊 Рейтинг блюд по реакциям гостей
  • 🔗 Автоматический redirect на Google Maps / TripAdvisor
  • 📧 Сбор email/телефона для повторных визитов
🚀
Позиционирование "Smart Brain": Конкуренты дают меню. Мы даём меню + аналитику + советы + автоматизацию. Это совершенно другой ценностный уровень и другой ценник.

10 Что добавить в ТЗ уже сейчас

Небольшие детали, которые стоит внести в основное ТЗ. Они не увеличивают срок разработки, но закрывают очевидные дыры от конкурентного анализа.

🏗 Архитектурные заготовки (добавить в схему БД)

📦 Таблица restaurants — 2 новых поля

Заготовка для POS-интеграции

Добавить поля: pos_system VARCHAR(20) DEFAULT 'manual' — значения: 'manual'|'iiko'|'rkeeper'|'poster', и pos_config JSONB DEFAULT NULL — зашифрованные API-ключи для Phase 3. В MVP не используются, но архитектура готова.

📦 Таблица menu_views — расширить

Добавить поле device_type

В таблице menu_views добавить device_type VARCHAR(20): 'ios'|'android'|'desktop'. Это бесплатная аналитика, которую никто из конкурентов не предлагает: "83% ваших гостей — iPhone пользователи".

⭐ Новая таблица dish_reactions

Быстрые реакции на блюда

Таблица: dish_reactions(id, dish_id, menu_view_id, reaction VARCHAR(10), created_at). Реакции: 'like'|'love'|'want'. Гость одним тапом отмечает блюдо. Ресторан видит рейтинг блюд по реакциям.

🎯 Продуктовые улучшения (незначительные)

🔥 Добавить в карточку блюда

"Хит" бейдж на основе аналитики

Если блюдо входит в топ-5 по просмотрам за последние 7 дней — автоматически показывать бейдж 🔥 "Популярное". Для Pro/Premium/LUXE. Это создаёт social proof без участия ресторана.

📱 UX улучшение

Последний выбранный язык — в localStorage

Запоминать выбранный гостем язык в localStorage. При повторном визите — автоматически открывать на том же языке. Мелочь, но создаёт ощущение "умного" продукта.

📊 Аналитика суперадмина

Карта Армении с активными ресторанами

В суперадмин-панели добавить простую карту (Mapbox/Leaflet) с точками активных ресторанов. Это красиво и показывает географический охват платформы. Быстро реализуется через координаты в таблице restaurants.

🌍 Туристический режим

Автоопределение языка по IP-геолокации

Добавить в флоу открытия меню: если IP-адрес иностранный — автоматически переключить на английский (или язык страны посетителя), а не ждать пока турист сам найдёт переключатель.

📋 Меню

Поддержка нескольких активных меню по времени

Сейчас: "только одно активное меню". Добавить: автоматическое переключение по расписанию. Например: завтрак 08:00-11:00 → меню "Завтраки", обед 12:00-16:00 → "Обеды", вечер 17:00-23:00 → основное меню. Поле active_from и active_until в таблице menus.

11 Выводы и стратегия

🎯 Главное преимущество

Мы — единственный локальный продукт с армянским языком, AMD, NFC, tourist mode и AI. Это нельзя скопировать быстро.

⏰ Окно возможностей

3-4 года до того, как западные игроки или r_keeper/iiko закроют армянский рынок. Нужно занять максимум ресторанов за этот период.

🔌 Следующий шаг

После MVP — интеграции с iiko и r_keeper. Это превратит нас из "альтернативы" в "обязательный слой" поверх любой POS-системы.
